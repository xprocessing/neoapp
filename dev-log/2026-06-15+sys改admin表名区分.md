# sys_* 表重命名为 admin_*，与 user_info 明确区分

## 修改范围
- `backend/src/main/resources/db/schema.sql` — 表名和分区注释
- `backend/src/main/java/com/neoapp/service/UserService.java` — 注释中的表名引用

## 重命名清单

| 旧表名 | 新表名 | 说明 |
|--------|--------|------|
| `sys_user` | `admin_user` | 后台管理员表 |
| `sys_role` | `admin_role` | 后台角色表 |
| `sys_user_role` | `admin_user_role` | 管理员角色关联表 |

## 表分区说明

```
admin_* 表 → 管理后台认证授权系统（面向后台管理员）
user_*  表 → 业务用户模块（面向 C 端用户）
```

- `admin_user` / `admin_role` / `admin_user_role` — 后台管理系统鉴权
- `user_info` / `user_wechat` / `user_address` — 业务用户相关

## 注意
当前后端 Java 代码中尚未创建 `admin_*` 表的实体类，这些表结构预留在 schema.sql 中供后续实现后台鉴权时使用。
