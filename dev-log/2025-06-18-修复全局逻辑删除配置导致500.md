# 2025-06-18 修复全局逻辑删除配置导致 500

## 问题
生产环境 `/api/admin/invites/rewards` 和 `/api/admin/points/logs` 返回 500 错误。

## 根因
`application.yml` 中配置了全局 `logic-delete-field: is_deleted`，MyBatis-Plus 对所有表查询自动添加 `WHERE is_deleted = 0` 条件。但以下表没有 `is_deleted` 列：
- `invite_reward_log`
- `point_log`
- `point_user`
- `member_user`

SQL 执行时报 `Unknown column 'is_deleted'` 语法错误。

## 修复
移除 `application.yml` 中的三行全局配置：
```yaml
logic-delete-field: is_deleted
logic-delete-value: 1
logic-not-delete-value: 0
```

各实体通过 `@TableLogic` 注解自主控制软删：
- `AdminUser.isDeleted` → `is_deleted` ✅
- `UserInfo.deleted` → `deleted` ✅
- `MemberPackage.deleted` → `deleted` ✅
- `TaskInfo.deleted` → `deleted` ✅
- `SysConfig.deleted` → `deleted` ✅
- `UserAddress.deleted` → `deleted` ✅

## 部署
重新编译 JAR 并上传到生产服务器，重启后端服务。
