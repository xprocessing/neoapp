# 2026-06-14 User模块 - CRUD + 注册登录

## 概述
在 neoapp 全栈项目基础上，新增完整的 user 业务模块，包括后端 API、管理端用户管理页面、Web端注册登录个人中心。

---

## 后端新增 / 修改

### 数据库 (schema.sql)
新增 3 张业务表：
- **user_info** — 用户主表（昵称/手机号/性别/邀请码/上下级关系/逻辑删除）
- **user_wechat** — 微信绑定表（openid/unionid/应用类型）
- **user_address** — 收货地址表（省市区/详细地址/默认标记）

### 实体类
- `entity/UserInfo.java` — 映射 user_info，使用 @TableLogic("deleted")
- `entity/UserWechat.java` — 映射 user_wechat
- `entity/UserAddress.java` — 映射 user_address，使用 @TableLogic("deleted")

### Mapper
- `mapper/UserInfoMapper.java`
- `mapper/UserWechatMapper.java`
- `mapper/UserAddressMapper.java`

### Service
- `service/UserInfoService.java` + `impl/UserInfoServiceImpl.java`
- `service/UserWechatService.java` + `impl/UserWechatServiceImpl.java`
- `service/UserAddressService.java` + `impl/UserAddressServiceImpl.java`
- `service/UserService.java` — 核心业务逻辑（注册/登录/分页查询/更新/删除 + 邀请码生成）

### DTO
- `dto/RegisterRequest.java` — 注册请求（手机号/密码/昵称/邀请码）
- `dto/LoginRequest.java` — 登录请求（手机号/密码）
- `dto/UserPageQuery.java` — 分页查询参数
- `dto/UserUpdateRequest.java` — 用户更新请求
- `dto/UserInfoVO.java` — 用户信息视图对象

### Controller
- `controller/UserAuthController.java` — `/api/auth/register`、`/api/auth/login`（公开接口）
- `controller/UserController.java` — `/api/user/info`（需认证）
- `controller/UserManageController.java` — `/api/admin/users/**` 增删改查（需认证）

### 安全配置
- `config/JwtAuthenticationFilter.java` — JWT 认证过滤器，从 Authorization header 解析 token
- `config/SecurityConfig.java` — 注入 JwtAuthenticationFilter，放行 `/api/auth/**`

### JWT 工具
- `util/JwtUtil.java` — 生成/解析/验证 JWT token

### 配置
- `application.yml` — 新增 jwt.secret 和 jwt.expiration 配置

## API 接口清单

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/auth/register | 用户注册 | 否 |
| POST | /api/auth/login | 用户登录，返回JWT | 否 |
| GET | /api/user/info | 获取当前用户信息 | 是 |
| GET | /api/admin/users | 分页查询用户列表 | 是 |
| GET | /api/admin/users/{id} | 用户详情 | 是 |
| PUT | /api/admin/users/{id} | 更新用户信息 | 是 |
| DELETE | /api/admin/users/{id} | 删除用户(逻辑删除) | 是 |
