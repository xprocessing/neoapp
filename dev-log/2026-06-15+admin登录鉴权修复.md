# Admin 登录鉴权修复

## 问题
`http://localhost:3004/dashboard` 点击"用户管理"后跳转到登录页。

## 根因分析

### 根因 1：登录使用假 Token
`admin/src/views/Login.vue` 使用 `mock_token_demo` 假 token，不是后端签发的真实 JWT。

- **Dashboard 能显示** — 页面数据全部硬编码，不调用任何 API
- **UserManage 页面跳登录** — `onMounted` 调用 `GET /api/admin/users`，后端 JWT 过滤器拒绝 `mock_token_demo`，返回 401，axios 拦截器收到 401 → 清 token → `window.location.href = '/login'`

### 根因 2：路由守卫缺失鉴权检查
`admin/src/router/index.js` 的 `beforeEach` 只设置了页面标题，未检查 `userStore.isLoggedIn`。任何用户都可直接访问受保护页面。

## 修复内容

### 1. admin/src/router/index.js — 添加路由鉴权
- 为 `/` 根路由添加 `meta: { requiresAuth: true }`
- `beforeEach` 中检查 `userStore.isLoggedIn`，未登录重定向到 `/login`

### 2. admin/src/views/Login.vue — 对接真实登录 API
- 表单字段 `username` → `phone`（手机号）
- 表单验证新增手机号格式校验 `/^1[3-9]\d{9}$/`
- 图标 `User` → `Phone`
- `handleLogin` 调用 `POST /api/auth/login`，获取真实 JWT token 存入 store
- 删除 mock 登录逻辑

## 测试验证
```bash
# 真实 token → 200 OK
curl -H "Authorization: Bearer <real_jwt>" http://localhost:8080/api/admin/users
# 假 token → 401（触发跳登录的旧行为）
curl -H "Authorization: Bearer mock_token_demo" http://localhost:8080/api/admin/users
```
